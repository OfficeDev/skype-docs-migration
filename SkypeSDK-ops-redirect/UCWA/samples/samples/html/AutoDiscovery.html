AutoDiscovery

AutoDiscovery is a behind the scenes programmatic task for discovering the correct location of the AutoDiscover service based on the provided sign-in address. Lync's AutoDiscovery process is well-defined, and begins with a request to https://lyncdiscoverinternal.domain (followed by https://lyncdiscover.domain), where domain is taken from the sign-in address.

Dependencies

  • GeneralHelper.js
  • Transport.js

Usage

In order to use the AutoDiscovery library a Transport will need to be properly constructed. The Transport libary requires the target domain for requests in order to correctly route messages. Technically that is all that is needed to construct the Transport library, but to faciliate cross-domain messaging the AutoDiscovery library will add an iframe to the DOM through the process of AutoDiscovery. AutoDiscovery begins with the startDiscovery(...) method which requires a domain, container, and callback. The domain is based on the sign-in address and will eventually lead to the user's home server. The container is a DOM element that will hold an iframe that will be dynamically injected into the DOM. This iframe is used exclusively by the Transport library to make cross-domain calls. The supplied callback will receive the a link to the AutoDiscovery Service root.
