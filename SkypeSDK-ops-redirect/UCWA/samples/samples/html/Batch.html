Batch

The Batch library allows the grouping of multiple requests into a single batch request send by the Transport. This enables the compacting of many cross-domain calls into a single request and response lowering overall number of remote calls. There is a risk of increasing response latency as the server will not send a response until every operation in the batch completes.

Dependencies

  • GeneralHelper.js
  • Cache.js
  • Transport.js

Usage

In order to construct the Batch library a Cache, a Transport, and a timer limit used to control processing are required. The batch library provides methods for queuing requests and flushing the queue even when not full. Queuing requests is accomplished by calling queueRequest(...) supplying a request object containing: URL, HTTP verb, data, accept type, and callback (to be executed when batch is completed). Processing outstanding requests is accomplished by calling processBatch() or once a request has been queued and the timer limit has expired (default: 3 seconds). Additionally, requests can be queued up until the 21st request at which point the current 20 requests will be processed and sent in a request.
